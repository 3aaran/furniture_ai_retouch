# 样式修改短口令

以后修改 Web 前端样式时，不需要每次重复完整规则。直接使用下面短口令即可。

## 通用口令

```txt
按项目样式边界规则，只改【页面/功能】的样式，实现【具体效果】。PC 和手机 H5 共用基础样式，手机端只做必要的响应式覆盖。不要改小程序。
```

示例：

```txt
按项目样式边界规则，只改资源库页面的样式，把资源卡片改得更紧凑。PC 和手机 H5 共用基础样式，手机端只做必要的响应式覆盖。不要改小程序。
```

## 只改 PC Web

```txt
按项目样式边界规则，只改【页面/功能】的 PC Web 样式，不影响手机 H5 和小程序。
```

## 只改手机 H5

```txt
按项目样式边界规则，只补【页面/功能】的手机 H5 适配，不重写整套页面样式，不影响 PC Web 和小程序。
```

## 同时改 PC 和手机 H5

```txt
按项目样式边界规则，调整【页面/功能】的 Web 样式。PC 和手机 H5 共用主要样式，只在必要位置添加 @media(max-width:860px) 的手机端覆盖。不要改小程序。
```

## 只改小程序

```txt
只改 miniapp-uni 小程序端的【页面/功能】样式，不要改 frontend Web 端样式。
```

## 规则含义

“按项目样式边界规则”默认包含：

- Web 端一套 React 页面，不拆 PC 和手机两套页面。
- PC Web 和手机 H5 共用基础样式。
- 手机端只写少量响应式覆盖。
- 小程序端只在 `miniapp-uni` 中修改。
- 修改某个页面时，优先改该页面对应 CSS。
- 不主动改 `legacy`、`overrides`、`body`、`html`、`.topMain` 等全局样式。
- 不新增宽泛全局选择器，例如 `.panel`、`.card`、`.toolbar`、`.modalMask`。
- 修改后运行 `npm --prefix frontend run build` 和 `npm --prefix frontend test`。

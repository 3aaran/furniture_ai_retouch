# 小程序图标、图片展示与加载策略设计

## 范围

仅修改 `miniapp-uni` 的前端页面、共享组件、静态图标和测试。保持既有页面结构、路由、鉴权和后端接口不变。

## 目标

1. 用与 Web 端同源的 Lucide 线框图标替换当前由 CSS 图形拼装的图标。
2. 顶栏在微信状态栏和右上角胶囊区域保持安全间距，头像、配额和菜单按钮不会互相挤压或被遮挡。
3. 任何业务图片在展示时不被容器裁切；需要检查细节的场景始终打开原图。
4. 浏览型列表优先请求缩略图，并采用懒加载和分页以减少首屏请求与内存占用。
5. 所有图标按钮具有清晰的边框、可点击尺寸与状态反馈。

## 图片数据策略

后端既有数据同时可提供原图 `url`（兼容 `imageUrl`、`resultUrl` 等）及缩略图 `thumbUrl`（兼容 `thumbnailUrl`）。小程序新增统一的图片选择辅助层：

| 场景 | 地址 | 显示方式 |
| --- | --- | --- |
| 资产库、历史记录、工作台最近生成、资源选择器 | `thumbUrl`，不存在时回退原图 | 懒加载，完整容器内展示 |
| 点击预览、对比、详情、下载、继续创作 | 原图 `url` | `uni.previewImage` 使用原图 |
| 上传后的产品原图、参考图 | 上传响应的原图 `url` | 完整展示，不裁切 |

图片容器使用比例盒与 `aspectFit`，不以 `aspectFill` 裁切主体。封面式裁切不在本次范围内使用。

## 交互与性能

列表按页追加：初始页只请求有限数量，触底时请求下一页；请求期间防止重复加载，最后一页显示结束状态。页面中所有浏览型 `image` 添加原生 `lazy-load`。预览仅在用户点击后加载原图。

## 图标与按钮

在小程序静态目录提供一组受控的 Lucide SVG 资源，并建立单一 `app-icon` 映射组件。页面继续使用现有图标名称，避免逐页复制图标标记。图标按钮统一为至少 `72rpx` 的可点击容器，使用可见边框、浅色底和明确的主/危险状态。

## 顶栏安全区

顶栏使用安全区变量和右侧最小保留空间。标题允许收缩但不覆盖头像；在可用宽度不足时优先隐藏或收缩配额文字，头像保留固定触控区域。头像的右边距不依赖截图中的系统胶囊位置。

## 验收与验证

- 静态测试验证不再保留 CSS 拼装图标、缩略图/原图选择逻辑、懒加载属性与分页状态。
- 既有小程序契约测试继续通过。
- 通过 HBuilderX 编译为 mp-weixin，检查无不支持的选择器或 WXML 解析错误。
- 在生成产物中确认关键页面图片标签与图标资源路径正确。
